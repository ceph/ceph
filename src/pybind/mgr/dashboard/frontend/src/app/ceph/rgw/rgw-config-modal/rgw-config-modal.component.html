Server-side Encryption
RGW SSE Configuration
Server-side encryption using Amazon S3-managed keys. Data is encrypted at rest using keys managed by the S3 service.
Server-side encryption using AWS Key Management Service (KMS). This option allows you to manage your encryption keys using AWS KMS.
Specify the provider for key management services. Common providers include AWS KMS, HashiCorp Vault, or other custom KMS solutions. This field is required.
Specify the authentication method to use when connecting to the key management service provider. This field is required.
Specify the secret engine to use when connecting to the key management service provider. For example, "kv" for Key-Value storage in HashiCorp Vault. This field is required.
The path in the key management service where the secret (encryption key) is stored. For HashiCorp Vault, this is the path to the key-value pair. This field is required.
(Optional) Specify the namespace within your key management service if applicable. Used to segregate secrets within the service.
The address of the HashiCorp Vault server. This is typically a URL such as https://vault.example.com. This field is required.
The token authentication method expects a Vault token to be present in a plaintext file. This field is required.
The SSL certificate in PEM format. This field is required.
The Client certificate in PEM format. This field is required.
The Client Private Key in PEM format. This field is required.
